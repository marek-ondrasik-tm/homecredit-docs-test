Aktuální stav rozhraní API snadno zjistíte pomocí zdroje [API state check resource](./get-health-using-get).

## Kontrola stavu API

Pokud jsou služby back-endu API v provozu, obdržíte stavový kód HTTP 204. V ostatních případech obdržíte jednu z následujících chyb:

| Stavový kód | Kód
| ----------- | --------------- |
| 500 | SERVICE_FAILED |
| 504 | SERVICE_TIMEOUT |

> Poznámka: zdroj DOES vyžaduje platný access_token v hlavičce HTTP požadavku
