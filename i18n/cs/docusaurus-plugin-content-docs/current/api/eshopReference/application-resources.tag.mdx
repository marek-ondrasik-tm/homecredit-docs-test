Služby, které umožňují zpracování *aplikace* během a
**objednávky** během celého jejich životního cyklu.

### Vytvořit aplikaci

Elementární tok začíná v okamžiku, kdy chce zákazník využít některou z platebních metod Home Creditu pro financování svého nákupu v partnerském e-shopu.

| Typ financování | Podporováno |
| ----------- | --------------- |
| `INSTALLMENT` | ANO |

Použijte správně zdroj `/financing/v1/applications`
vyplněnou žádostí, která obsahuje alespoň následující údaje

- Údaje o zákazníkovi není nutné zasílat, nicméně jejich zaslání je velmi doporučeno - zjednoduší to klientovi proces podání žádosti
  - Jméno (jméno a příjmení nebo celé jméno)
  - E-mail
  - Číslo mobilního telefonu
  - Adresa (pole)
    - Město
    - Ulice
    - Číslo ulice
    - PSČ
    - Typ adresy

- Údaje o objednávce
  - Číslo
  - Celková cena (včetně DPH)
  - Položky objednávky (pole, je vyžadována alespoň jedna položka)
    - Kód
    - Název položky objednávky
    - Celková cena (všechny kusy včetně DPH)

- Typ financování (vždy `INSTALACE`)
- Údaje o typu financování
  - *Nastavení splátky - preferovaná výše splátky
  - *Nastavení splátek - preferovaná výše zálohy
  - *Nastavení splátky - preferovaný kód produktu programu

- Další informace nezbytné pro pokračování a dokončení procesu
  - Vaše adresa URL, na kterou má být zákazník přesměrován po schválení žádosti
  - Vaše adresa URL, kam by měl být zákazník přesměrován po zamítnutí žádosti
  - Adresa URL koncového bodu oznámení, na který chcete, abychom zasílali oznámení o konkrétních aktualizacích aplikace (v případě, že partner používá notifikátor)

> * = Musí být správně vyplněno v případě, že zákazník již použil splátkovou kalkulačku vloženou na vašich webových stránkách a zvolil preferovaný splátkový program. Více informací o možnosti vložení splátkové kalkulačky na vaše webové stránky naleznete v části Zdroje splátkové kalkulačky. V opačném případě ponechte prázdné.

Pokud byl požadavek úspěšně zpracován, obdržíte v odpovědi stavový kód HTTP 201 a objekt aplikace s minimálně následujícími informacemi:

- Unikátní ID aplikace
- Stav aplikace
- Podstav aplikace
- Adresa URL přesměrování brány, kam máte zákazníka po zpracování odpovědi přesměrovat

> Můžete také obdržet informaci o původně požadované **platbě dolů** - to
může být později zákazníkem změněna v průběhu procesu finalizace financování
žádosti. V takových případech doporučujeme zkontrolovat konečnou **dodatečnou platbu**
schváleného úvěru (po schválení žádosti) prostřednictvím `/financování/v1/žádosti/{Id žádosti}`.

Pokud byla žádost **rozpoznána jako duplicita**

- a dříve vytvořená `Žádost` je zrušena (`Žádost.stav = ZRUŠENÁ`a
`Application.stateReason = CANCELLED_NOT_PAID`), je vytvořena nová `Application`, vy
obdržíte stavový kód HTTP 201 a objekt `Application` s výše uvedenými informacemi (minimálně) v odpovědi
- a dříve vytvořená `Application` je stále nová (tj. bez zaznamenané akce klienta), žádná `Application` není vytvořena, obdržíte stavový kód HTTP 200 a objekt `Application` vytvořený při prvním souvisejícím úspěšném volání zdroje.
- a dříve vytvořená `Aplikace` již není zrušena ani nová, obdržíte stavový kód HTTP 422 s kódem chyby `DUPLICATE_ORDER_NUMBER`.

V případě chyby obdržíte jeden z následujících výsledků:

| Stavový kód | Kód | Vysvětlení |
| ----------- | --------------- | -------------------- |
| 400 | INVALID_REQUEST | Požadavek nebyl správně naformátován (chybná syntaxe požadavku, příliš velká velikost atd.) |
| 422 | UNPROCESSABLE | Požadavek byl dobře zformulován, ale nebylo možné jej následovat kvůli sémantickým chybám |
| 422 | DUPLICATE_ORDER_NUMBER | Vytvoření žádosti s duplicitním číslem objednávky není povoleno |
| 500 | INTERNAL_SERVER_ERROR | Nastala neočekávaná podmínka a není vhodná žádná konkrétnější zpráva |

### Detail žádosti

Pokud potřebujete zjistit aktuální informace o konkrétní *aplikaci* - především její stav
a další podrobnosti - použijte zdroj `/financing/v1/applications/{applicationId}`, kde `{applicationId}`
je ID aplikace, které jste obdrželi v odpovědi na `/financing/v1/applications` po jejím vytvoření

| Typ financování | Podporováno |
| ----------- | --------------- |
| `INSTALLMENT` | YES |

Pokud byl požadavek úspěšně zpracován, obdržíte v odpovědi stavový kód HTTP 200 a objekt aplikace s minimálně následujícími informacemi:

- Jedinečné ID aplikace
- Typ aplikace
- Stav aplikace
- Podstav aplikace
- Adresa URL přesměrování brány

> Můžete také obdržet informaci o aktuální **dodatečné platbě**, která je užitečná, když
potřebujete zjistit, zda byl zákazníkovi schválen úvěr s jinou zálohou
než bylo původně nastaveno ve vaší vestavěné splátkové kalkulačce.

V ostatních případech obdržíte jednu z následujících chyb:

| Stavový kód | Kód | Vysvětlení |
| ----------- | --------------- | -------------------- |
| 400 | INVALID_REQUEST | Požadavek nebyl správně naformátován (chybná syntaxe požadavku, příliš velká velikost atd.) |
| 404 | APPLICATION_NOT_FOUND | Požadovaný prostředek nebyl nalezen |
| 422 | NOT_SUPPORTED | Zdroj není pro daný typ financování podporován |
| 422 | UNPROCESSABLE | Požadavek byl správně formulován, ale nebylo možné jej splnit kvůli sémantickým chybám |
| 500 | INTERNAL_SERVER_ERROR | Nastal neočekávaný stav a není vhodná žádná konkrétnější zpráva |

### Zrušit žádost

Pokud je potřeba zcela zrušit objednávku zákazníka, je nutné zrušit příslušnou objednávku
*aplikaci*. K tomuto účelu použijte `/financing/v1/applications/{applicationId}/cancel`
zdroj, kde `{applicationId}` je ID *aplikace*, které jste obdrželi v odpovědi na příkaz
`/financing/v1/applications` po jejím vytvoření.

| Typ financování | Podporováno |
| ----------- | --------------- |
| `INSTALLMENT` | YES |


| Stav žádosti | Dostupnost |
| ----------- | --------------- |
| `ODMÍTNUTO` | ANO |
| `ZRUŠENO` | ANO |
| `PROCESSING` | ANO |
| `PŘIPRAVENO` | ANO |

Pokud byla žádost úspěšně zpracována, ale žádost již byla zaplacena
(`Application.stateReason = READY_PAID`), obdržíte stavový kód HTTP **202 ACCEPTED**
což znamená, že byl zahájen proces zrušení *žádosti*. Mělo by to být
dokončen po provedení několika ručních úkonů v back office.

V ostatních případech, kdy je žádost úspěšně zpracována, obdržíte stavový kód HTTP **200 OK**.

V odpovědi také obdržíte minimálně následující informace o (brzy) zrušené žádosti:

- Jedinečné ID žádosti
- Typ žádosti
- Stav žádosti
- Podstav žádosti

V ostatních případech se zobrazí jedna z následujících chyb:

| Stavový kód | Kód | Vysvětlení |
| ----------- | --------------- | -------------------- |
| 400 | INVALID_REQUEST | Požadavek nebyl správně naformátován (chybná syntaxe požadavku, příliš velká velikost atd.) |
| 404 | OBJECT_NOT_FOUND | Požadovaný prostředek nebyl nalezen |
| 422 | NOT_SUPPORTED | Zdroj není pro daný typ financování podporován |
| 422 | UNPROCESSABLE | Požadavek byl správně formulován, ale nebylo možné jej splnit kvůli sémantickým chybám |
| 422 | APPLICATION_REJECTED | Při pokusu o zrušení již `REJECTED` `Aplikace`, což není povoleno |
| 500 | INTERNAL_SERVER_ERROR | Nastala neočekávaná podmínka a není vhodná žádná konkrétnější zpráva |

### Změna pořadí aplikace

Pokud dojde k nějakým změnám v rámci pořadí, které mohou mít důležitý dopad na
na proces splácení po jeho podání, použijte `/financing/v1/applications/{applicationId}/order`
abychom mohli vše správně upravit. `{applicationId}` je ID žádosti
které jste obdrželi v odpovědi `/financing/v1/applications` po jejím vytvoření.

| Typ financování | Podporováno |
| ----------- | --------------- |
| `INSTALLMENT` | YES |

| Stav žádosti | Dostupnost |
| ----------- | --------------- |
| `ODMÍTNUTO` | NE |
| `ZRUŠENO` | NE
| `PROCESSING` | ANO |
| `READY` | ANO* (viz poznámka o dostupnosti níže) |

Použijte zdroj `/financing/v1/applications/{applicationId}/order` se správně vyplněnými údaji
požadavkem, který obsahuje (pouze) změněná data (prostředek umožňuje částečné vložení).

> **POZNÁMKA K DOSTUPNOSTI**: Platný objekt požadavku obsahuje pouze číslo objednávky (`order.number`)
a/nebo jednu položku v poli `order.variableSymbols[]`. Podporovaný `stateReason` stavu `state READY`
jsou `READY_TO_SHIP`, `READY_SHIPPED`, `READY_DELIVERING` (`state PROCESSING` je podporován bez omezení).

Pokud byl požadavek úspěšně zpracován, zobrazí se stavový kód HTTP 200 a
objekt informace o změněné objednávce a základní informace o příslušné objednávce
žádosti (ID, stav, podstav) s minimálně následujícími informacemi v odpovědi:

- Číslo objednávky
- Celková cena objednávky (včetně DPH)
- Položky objednávky (pole)
  - Název položky objednávky
  - Celková cena (všechny kusy včetně DPH)
  - Stav položky objednávky
- Stav objednávky
- Základní informace o aplikaci
  - ID
  - Stav aplikace
  - Podstata aplikace

V ostatních případech se zobrazí jedna z následujících chyb:

| Stavový kód | Kód | Vysvětlení |
| ----------- | --------------- | -------------------- |
| 400 | INVALID_REQUEST | Požadavek nebyl správně naformátován (chybná syntaxe požadavku, příliš velká velikost atd.) |
| 404 | OBJECT_NOT_FOUND | Požadovaný prostředek nebyl nalezen |
| 422 | NOT_SUPPORTED | Zdroj není pro daný typ financování podporován |
| 422 | UNPROCESSABLE | Požadavek byl správně formulován, ale nebylo možné jej splnit kvůli sémantickým chybám |
| 422 | APPLICATION_REJECTED | Při pokusu o zrušení již `REJECTED` `Aplikace`, což není povoleno |
| 500 | INTERNAL_SERVER_ERROR | Nastal neočekávaný stav a není vhodná žádná konkrétnější zpráva |
