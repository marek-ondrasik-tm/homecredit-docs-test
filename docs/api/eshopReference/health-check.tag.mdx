To easily find out current API state use the [API state check resource](./get-health-using-get).

## API health check

If the API back-end services is up and running, you receive HTTP status code 204. In other cases, you receive one of the following errors:

| Status code | Code            |
| ----------- | --------------- |
| 500         | SERVICE_FAILED  |
| 504         | SERVICE_TIMEOUT |

> Note: resource DOES require valid access_token in the HTTP header of the request
